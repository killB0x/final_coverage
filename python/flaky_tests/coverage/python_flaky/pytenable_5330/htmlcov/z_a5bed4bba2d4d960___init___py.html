16pyTenable is intended to be a pythonic interface into the Tenable application 

17APIs. Further by providing a common interface and a common structure between 

18all of the various applications, we can ease the transition from the vastly 

19different APIs between some of the products. 

20 

21- Issue Tracker: https://github.com/tenable/pyTenable/issues 

22- Github Repository: https://github.com/tenable/pyTenable 

23 

24Installation 

25------------ 

26 

27To install the most recent published version to pypi, its simply a matter of 

28installing via pip: 

29 

30.. code-block:: bash 

31 

32 pip install pytenable 

33 

34If your looking for bleeding-edge, then feel free to install directly from the 

35github repository like so: 

36 

37.. code-block:: bash 

38 

39 pip install git+git://github.com/tenable/pytenable.git#egg=pytenable 

40 

41Getting Started 

42--------------- 

43 

44Lets assume that we want to get the list of scans that have been run on our 

45Tenable.io application. Performing this action is as simple as the following: 

46 

47.. code-block:: python 

48 

49 from tenable.io import TenableIO 

50 tio = TenableIO('TIO_ACCESS_KEY', 'TIO_SECRET_KEY') 

51 for scan in tio.scans.list(): 

52 print('{status}: {id}/{uuid} - {name}'.format(**scan)) 

53 

54Getting started with Tenable.sc is equally as easy: 

55 

56.. code-block:: python 

57 

58 from tenable.sc import TenableSC 

59 sc = TenableSC('SECURITYCENTER_NETWORK_ADDRESS') 

60 sc.login('SC_USERNAME', 'SC_PASSWORD') 

61 for vuln in sc.analysis.vulns(): 

62 print('{ip}:{pluginID}:{pluginName}'.format(**vuln)) 

63 

64For more detailed information on whats available, please refer to the navigation 

65section for the Tenable application you're looking 

66 

67Logging 

68------- 

69 

70Enabling logging for pyTenable is a simple matter of enabling debug logs through 

71the python logging package. An easy example is detailed here: 

72 

73.. code-block:: python 

74 

75 import logging 

76 logging.basicConfig(level=logging.DEBUG)
