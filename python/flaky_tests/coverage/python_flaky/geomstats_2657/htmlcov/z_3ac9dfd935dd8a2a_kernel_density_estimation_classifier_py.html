10class KernelDensityEstimationClassifier(RadiusNeighborsClassifier): 

11 """Classifier implementing the kernel density estimation on manifolds. 

12 

13 The kernel density estimation classifier classifies the data according to 

14 a kernel density estimation of each dataset on the manifold. The density 

15 estimation is performed using radial kernel functions: the distance 

16 is the only geometrical tool used to estimate the density on the manifold. 

17 This classifier inherits from the radius neighbors classifier of the 

18 scikit-learn library, we expect the classifier presented here to be easier 

19 to use on manifolds. 

20 Compared with the radius neighbors classifier, we force the 

21 parameter 'algorithm' to be equal to 'brute' in order to 

22 be compatible with any metric. 

23 We also changed some default values of the scikit-learn algorithm in order 

24 to take into account every point of the dataset during the kernel density 

25 estimation, i.e. the default value of the parameter 'radius' is set to 

26 infinity instead of 1 and the default value of the parameter 'weight' is 

27 set to 'distance' instead of 'uniform'. 

28 Our main contribution is a greater choice of kernel functions, 

29 see the radial_kernel_functions.py file in the learning directory. 

30 The radial kernel functions are now easier to define by a user: 

31 the input data should be an array of distances instead of an array of 

32 arrays. Moreover the new parameter 'bandwidth' of our classifier can be 

33 used to adapt the kernel function to the size of the dataset. 

34 The scikit-learn library also provides a kernel density estimation tool 

35 (see sklearn.neighbors.KernelDensity), however this algorithm is not built 

36 as a classifier and is not available with all metrics. 

37 

38 Parameters 

39 ---------- 

40 radius : float, optional (default = inf) 

41 Range of parameter space to use by default. 

42 kernel : string or callable, optional (default = 'distance') 

43 Kernel function used in prediction. Possible values: 

44 - 'distance' : weight points by the inverse of their distance. 

45 In this case, closer neighbors of a query point will have a 

46 greater influence than neighbors which are further away. 

47 - 'uniform' : uniform weights. All points in each neighborhood 

48 are weighted equally. 

49 - [callable] : a user-defined function which accepts an 

50 array of distances, and returns an array of the same shape 

51 containing the weights. 

52 bandwidth : float, optional (default = 1.0) 

53 Bandwidth parameter used for the kernel. The kernel parameter is 

54 used if and only if the kernel is a callable function. 

55 p : integer, optional (default = 2) 

56 Power parameter for the 'minkowski' string distance. 

57 When p = 1, this is equivalent to using manhattan_distance (l1), 

58 and euclidean_distance (l2) for p = 2. 

59 For arbitrary p, minkowski_distance (l_p) is used. 

60 distance : string or callable, optional (default = 'minkowski') 

61 The distance metric to use. 

62 The default distance is minkowski, and with p=2 is equivalent to the 

63 standard Euclidean distance. 

64 See the documentation of the DistanceMetric class in the scikit-learn 

65 library for a list of available distances. 

66 If distance is "precomputed", X is assumed to be a distance matrix and 

67 must be square during fit. 

68 outlier_label : {manual label, 'most_frequent'}, optional (default = None) 

69 Label for outlier samples (samples with no neighbors in given radius). 

70 - manual label: str or int label (should be the same type as y) 

71 or list of manual labels if multi-output is used. 

72 - 'most_frequent' : assign the most frequent label of y to outliers. 

73 - None : when any outlier is detected, ValueError will be raised. 

74 distance_params : dict, optional (default = None) 

75 Additional keyword arguments for the distance function. 

76 n_jobs : int or None, optional (default = None) 

77 The number of parallel jobs to run for neighbors search. 

78 ``None`` means 1; ``-1`` means using all processors. 

79 

80 Attributes 

81 ---------- 

82 classes_ : array-like, shape=[n_classes,] 

83 Class labels known to the classifier. 

84 effective_metric_ : string or callable 

85 The distance metric used. It will be same as the `distance` parameter 

86 or a synonym of it, e.g. 'euclidean' if the `distance` parameter set to 

87 'minkowski' and `p` parameter set to 2. 

88 effective_metric_params_ : dict 

89 Additional keyword arguments for the distance function. 

90 For most distances will be same with `distance_params` parameter, 

91 but may also contain the `p` parameter value if the 

92 `effective_metric_` attribute is set to 'minkowski'. 

93 outputs_2d_ : bool 

94 False when `y`'s shape is [...,] or [..., 1] during fit, 

95 otherwise True. 

96 

97 References 

98 ---------- 

99 This algorithm uses the scikit-learn library: 

100 https://scikit-learn.org/stable/modules/generated/ 

101 sklearn.neighbors.RadiusNeighborsClassifier.html 

102 """
