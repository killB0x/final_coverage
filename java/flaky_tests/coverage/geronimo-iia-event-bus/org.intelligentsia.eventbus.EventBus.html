 /**
 34   
  * An EventBus is a simple pattern to promote loose coupling between various
 35   
  * components. An event bus can be used in place of a traditional listener or
 36   
  * observer pattern, and is useful when ties between multiple components become
 37   
  * too complicated to track.
 38   
  * <p>
 39   
  * A traditional use case for an event bus is a Swing based application, where
 40   
  * multiple actions and listeners are configured to capture the various events
 41   
  * of the application, such as mouse clicks, window events, data loading events,
 42   
  * etc. Swing promotes a one-to-one mapping between listener/listenee
 43   
  * components, and as such, it can become difficult to configure the various
 44   
  * listeners without tightly coupling all the various componenets together.
 45   
  * <p>
 46   
  * With an event bus, events can be published to the bus and any class can be
 47   
  * configured to listen for such events. Thus, each individual component only
 48   
  * needs to be tightly coupled with the event bus, and it can then receive
 49   
  * notifications about events that it cares to know about.
 50   
  * <p>
 51   
  * The event bus pattern has a simple interface, with a subscribe/publish type
 52   
  * model. Any object can be subscribed to the event bus, but to received
 53   
  * messages from the bus, the object must have its methods annotated with the
 54   
  * {@link EventHandler} annotation. This annotation marks the methods of the
 55   
  * subscriber class which should be used to receive event bus events.
 56   
  * <p>
 57   
  * A published event has the potential to be vetoed and thus not propagated to
 58   
  * other non-vetoing subscribers. This is accomplished by setting the
 59   
  * {@link EventHandler#canVeto()} property to true and throwing a
 60   
  * {@link VetoException} when the method is called from the EventBus. The event
 61   
  * bus will note the veto and not relay the message to the subscribers, but will
 62   
  * instead send a {@link VetoEvent} out on the bus indicating that the published
 63   
  * event has been vetoed.
 64   
  *
 65   
  * @author Adam Taft
 66   
  * @author <a href="mailto:jguibert@intelligents-ia.com" >Jerome Guibert</a>
 67   
  */
